Installation

FormPath JS is designed to work out of the box. No bundlers, no complex build steps—just high-performance form management.

Use via cdnjs (Recommended)

Paste this into your HTML <head> or before the closing </body> tag:

<script src="https://cdnjs.cloudflare.com/ajax/libs/FormPath-js/3.2.0/FormPath.min.js"></script>

Usage in any static HTML 

<script>
  // It initializes automatically on DOMReady
  const ff = new FormPath({
    onChange: (data) => console.log('Live Data:', data)
  });
</script>

Form Discovery

By default, FormPath JS performs a global scan of the DOM and observes every <form> tag it finds. However, you can explicitly target specific forms to optimize performance or separate concerns.

Default Behavior

Automatically attaches listeners to all forms on the page.

new FormPath();

Targeted Selection

Provide an array of IDs to limit the scope of the library.

new FormPath({
  formIds: ['user-settings', 'billing-info']
});

Path Mapping Logic

The library uses the name attribute of your inputs to build a deep JSON object. It supports dot notation for objects and bracket notation for arrays natively.

Input Attributes

  • name="user.id"
  • name="user.profile.bio"
  • name="tags[0]"
  • name="items[0].price"

Resulting JSON

{
  "user": {
    "id": "123",
    "profile": { "bio": "..." }
  },
  "tags": ["..."],
  "items": [{ "price": "..." }]
}

IndexedDB Engine

Unlike LocalStorage which has a 5MB limit and blocks the main thread, FormPath uses IndexedDB for asynchronous, high-capacity persistence.

Object Store Architecture

The database FormPathDB contains a single object store formData where each entry is indexed by its HTML Form ID.

Key: "shipping-form"
Value: { "addr": "..." }
Key: "profile-form"
Value: { "name": "..." }

Data Access

Retrieval is always asynchronous to ensure compatibility with the IndexedDB engine.

// Get all data from IndexedDB
const allData = await ff.getAllData();

// Access specific section
const profile = allData['profile-form'];

Clearing Data

Utility

You can programmatically clear the stored data. This is essential for "Reset" buttons or clearing cache after a successful form submission.

Targeted Form Clear

Deletes specific entries from IndexedDB by providing their Form IDs.

ff.clearData(['registration-form']);

Global Wipe

Clears every single piece of data stored in the FormPath object store.

ff.clearData(); // No arguments clears all
